# 页面管理功能使用说明

## 功能概述

现在你可以在后台管理系统中创建和管理自定义页面，例如"关于我们"、"隐私政策"、"服务条款"等。

## 如何使用

### 1. 进入后台管理

- 在网站任意页面按键盘上的 `admin` 五个字母（按顺序输入）
- 或者在浏览器地址栏输入对应的后台路径

### 2. 创建新页面

1. 点击左侧导航栏的 **"页面管理"** 标签
2. 点击右上角的 **"创建页面"** 按钮
3. 填写页面信息：
   - **页面标题**：显示在页面顶部的标题
   - **URL路径 (slug)**：页面的访问地址，只能包含小写字母、数字和连字符
     - 例如：`about-us` → 访问地址为 `/page/about-us`
   - **页面内容**：支持纯文本和HTML标签
     - 可以使用 `<h2>标题</h2>`、`<p>段落</p>`、`<ul><li>列表项</li></ul>` 等HTML标签
4. 勾选设置选项：
   - **发布页面**：勾选后用户才能看到这个页面
   - **在页脚导航显示**：勾选后页面链接会出现在网站页脚的"客户服务"栏目中
5. 点击 **"保存更改"** 按钮

### 3. 编辑现有页面

1. 在页面列表中找到要编辑的页面
2. 点击右侧的 **"编辑"** 按钮
3. 修改内容后保存

### 4. 删除页面

1. 在页面列表中找到要删除的页面
2. 点击右侧的 **"删除"** 按钮
3. 确认删除（注意：删除后无法恢复）

## 页面状态说明

- **已发布**：绿色标签，用户可以访问该页面
- **草稿**：灰色标签，页面存在但用户无法访问

## 常用页面示例

以下是一些常见的自定义页面建议：

| 页面标题 | URL路径 (slug) | 说明 |
|---------|---------------|------|
| 关于我们 | about-us | 公司介绍、品牌故事 |
| 隐私政策 | privacy-policy | 隐私保护声明 |
| 服务条款 | terms-of-service | 使用条款和协议 |
| 品牌故事 | brand-story | 品牌发展历程 |
| 合作伙伴 | partners | 合作商家信息 |
| 联系方式 | contact-info | 详细联系信息 |

## HTML内容示例

### 简单文本页面
```html
<h2>欢迎来到我们的商店</h2>
<p>我们致力于提供优质的产品和服务。</p>
<p>如有任何问题，请随时联系我们。</p>
```

### 带列表的页面
```html
<h2>我们的优势</h2>
<ul>
  <li>100%正品保证</li>
  <li>快速配送服务</li>
  <li>7天无理由退换</li>
  <li>24小时在线客服</li>
</ul>
```

### 多段落结构化页面
```html
<h2>公司简介</h2>
<p>公司成立于2020年，专注于情趣用品领域...</p>

<h2>我们的使命</h2>
<p>为客户提供安全、优质、私密的购物体验...</p>

<h2>联系我们</h2>
<p>Email: support@example.com</p>
<p>工作时间: 周一至周五 9:00-18:00</p>
```

## 国际化支持

- 创建的自定义页面会自动支持网站的多语言功能
- 当用户切换语言时，页面标题和内容会自动翻译
- 翻译通过LibreTranslate API实时进行

## 访问自定义页面

用户可以通过以下方式访问自定义页面：

1. **页脚导航**：如果勾选了"在页脚导航显示"，页面链接会出现在网站底部
2. **直接访问**：输入完整URL地址，例如 `https://yoursite.com/page/about-us`

## 注意事项

1. **URL路径唯一性**：每个页面的slug必须唯一，不能重复
2. **内容安全**：HTML内容会直接渲染，请确保内容安全可靠
3. **发布状态**：只有勾选"发布页面"的页面才能被用户访问
4. **数据备份**：页面数据保存在浏览器localStorage中，建议定期在"系统设置"中下载备份

## 技术细节

- 页面数据存储在 `StoreConfig.customPages` 中
- 每个页面包含以下字段：
  - `id`: 唯一标识符
  - `title`: 页面标题
  - `slug`: URL路径
  - `content`: 页面内容（支持HTML）
  - `isPublished`: 是否发布
  - `showInFooter`: 是否在页脚显示
  - `createdAt`: 创建时间
  - `updatedAt`: 更新时间
